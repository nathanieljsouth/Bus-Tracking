Decisions

I had to make quite a few design and functional decisions throughout the project. These are listed here so the reasons are not forgotten.


I decided to only store the most recent location of each bus route. This was because I only need the most recent location to enable the user app to work, and also to restrict the amount of storage the app needs. Also, this prevents the bus driver's privacy from being invaded by recording all the bus' movements permanently


I decided to display a 'confidence level' for the estimated time of arrival of the bus based upon how old the location data for the bus was. This was because making an estimated on old data is very unreliable and the user needs to know whether they can trust the estimate.


I decided to not trust any location data older than 20 minutes. If the bus location data is older than 20 minutes I display a message saying the bus hasn't checked in recently.


I decided to not display a scheduled time that the bus is due to arrive at a stop. This was because the users are interested in when the bus is going to arrive, not when it was meant to arrive.


I decided to create a 'virtual bus' page that allows a tester to move a bus on a map. This was because testing using real GPS locations is not practical (it would involve driving around) and testing using latitude / longitude coordinates is not easy to understand.


I decided to ignore the fact that Google may calculate arrival times using a route that does not match the bus route. This was because any estimate from Google will always be at least as quick as the bus route so all results will err on the side of being too early, so users won't miss their bus.


I decided to store all the routes and their stops in a single JSON file. This was because separating them out was actually more work and made it easier to make mistakes, and JSON is great for structures like this.


I decided to specify the centre of the map for each route separately, along with the zoom value. This was because each route could cover different size areas, and I wanted to always centre the map around the route and not around the user.


I decided to remove the links I added to the pages to zoom the map in and out, and show the zoom level. THis is because the map itself has zoom controls and people are used to pinching on mobile devices to zoom the map.
